# 텔레그램 발송 양식 정의

TrendSurfer의 텔레그램 알림 양식 정의 문서입니다. 다른 모듈·프로젝트에서 동일한 메시지를 재현할 수 있도록 API 스펙과 본문 포맷을 정리합니다.

> 정의 위치: `backend/app/services/notifier.py` (`Notifier` 클래스, 싱글톤 인스턴스 `notifier`)

---

## 1. 전송 방식 (Telegram API 호출 스펙)

`send_message()` — `notifier.py:39`

| 항목 | 값 |
|------|----|
| **엔드포인트** | `https://api.telegram.org/bot{token}/sendMessage` |
| **메서드** | `POST` (JSON body) |
| **parse_mode** | `"HTML"` (기본값) |
| **timeout** | 10초 |
| **길이 제한** | 본문 4000자 초과 시 잘라내고 `\n\n(메시지 길이 초과로 일부 생략)` 추가 |
| **환경변수** | `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` |

**payload 구조:**

```json
{
  "chat_id": "<TELEGRAM_CHAT_ID>",
  "text": "<메시지 본문>",
  "parse_mode": "HTML"
}
```

- 강조 표기는 HTML 태그 `<b>...</b>` 사용
- 헤더 구분은 이모지 사용 (`📊` / `📅` / `✅`)

---

## 2. 신호 리포트 양식 (`send_signal_report`, `notifier.py:79`)

- 강도순 상위 **20개**(`max_display`)만 표시, 나머지는 `... 외 M개 종목`
- 종가는 천 단위 콤마(`{close:,}`)
- 각 종목 블록은 2줄 구성 (이름줄 + 상세줄, 상세줄은 공백 3칸 들여쓰기)

### 신호가 있을 때

```
📊 <b>TrendSurfer 신호 리포트</b>
📅 {target_date} | {N}개 종목 발견

1. <b>{종목명}</b> ({종목코드})
   종가 {종가:,} | 강도 {강도}% | 거래대금 {거래대금}억 {스테이지이모지}
2. <b>{종목명}</b> ({종목코드})
   종가 {종가:,} | 강도 {강도}% | 거래대금 {거래대금}억 {스테이지이모지}
...

... 외 {M}개 종목
```

### 신호가 없을 때

```
📊 <b>TrendSurfer 신호 리포트</b>
📅 {target_date}

오늘 조건을 충족하는 종목이 없습니다.
```

## 3. 일일 요약 양식 (`send_daily_summary`, `notifier.py:131`)

- `market_status`가 없으면 "시장:" 줄은 생략

```
✅ <b>일일 루틴 완료</b>
📅 {target_date}

시장: KOSPI {🟢|🔴} | KOSDAQ {🟢|🔴}
신호: {signal_count}개 종목 발견
```

## 4. EMA 스테이지 이모지 매핑 (`_stage_emoji`, `notifier.py:165`)

| stage | 이모지 | 의미 |
|-------|--------|------|
| 1 | `🟢S1` | 안정 상승 |
| 2 | `🟡S2` | 하락 변화1 |
| 3 | `🟠S3` | 하락 변화2 |
| 4 | `🔴S4` | 안정 하락 |
| 5 | `🟠S5` | 상승 변화1 |
| 6 | `🟡S6` | 상승 변화2 |
| 그 외(0/None) | `⚪S0` | - |

## 5. 재사용 방법

### 같은 프로젝트 내 (권장)

새로 구현하지 말고 기존 싱글톤을 그대로 import 합니다.

```python
from app.services.notifier import notifier

notifier.send_message("<b>제목</b>\n내용")               # 임의 메시지
notifier.send_signal_report(signals, "2026-06-18")      # 신호 리포트
notifier.send_daily_summary("2026-06-18", signal_count=5)
```

### 외부 프로젝트에서 재현

위 **1번 API 스펙**과 **2~4번 본문 포맷**을 그대로 옮기면 동일한 메시지가 생성됩니다.
핵심 규칙:

1. `parse_mode="HTML"`
2. 강조는 `<b>...</b>`
3. 이모지로 헤더 구분 (`📊` / `📅` / `✅`)
4. 본문 4000자 컷
