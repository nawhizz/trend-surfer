# 제품 요구사항 문서 (PRD): TrendSurfer (트렌드서퍼) v1.0

## 1. 개요 (Overview)
- **한 줄 요약:** 퀀트 지표 기반으로 시장의 추세를 분석하고, 직장인을 위해 매매 타이밍을 자동으로 포착/알림해주는 자동매매 솔루션.
- **프로젝트 비전:** "시장 파동의 흐름을 타고, 데이터로 수익을 확정한다." (감에 의존하지 않는 객관적 투자 시스템 구축)
- **작성자:** TrendSurfer Team
- **버전:** v1.0 (MVP)

## 2. 배경 및 문제 정의 (Background & Problem Statement)
- **문제 상황:** 직장인은 업무 시간(09:00~18:00) 중 주식 시장 변동에 실시간으로 대응하기 어렵습니다. 이로 인해 매수/매도 타이밍을 놓치거나, 뒤늦은 뇌동매매로 손실을 보는 경우가 많습니다.
- **사용자에게 미치는 영향:** 수익 기회 상실, 투자 실패로 인한 스트레스, 본업 집중도 저하.
- **해결 방안:** 시스템이 장중(12~14시) 유망 종목을 선별하여 알림을 주고, 장 마감 후 데이터를 분석하여 다음 날 전략을 수립하는 과정을 **자동화**합니다.

## 3. 목표 및 성공 지표 (Goals & Success Metrics)
- **Objective 1 (시스템 안정성):** 데이터 수집 및 가공 파이프라인의 완전 자동화.
  - **KR 1.1:** 코스피/코스닥 전 종목(약 2,500개) 일별 데이터 수집 성공률 99% 달성.
  - **KR 1.2:** 장중(12:00~14:00) 전략 스크리닝 실행 및 알림 발송 지연 시간 1분 이내.
- **Objective 2 (가치 검증):** 객관적 지표에 근거한 투자 기회 제공.
  - **KR 2.1:** 시스템이 포착한 종목 중, 익일 상승 종목 비율(Hit Rate) 측정 환경 구축.

## 4. 사용자 페르소나 및 시나리오 (User Personas & Scenarios)
- **핵심 페르소나:** '김대리' (32세, IT 기업 직장인)
  - **특징:** 파이썬 기초 지식이 있음. 퇴근 후 차트를 보며 공부하지만, 장중에는 회의 때문에 MTS를 볼 수 없음. 퀀트 투자에 관심이 많음.
- **사용 시나리오:**
  1.  **출근 전:** n8n이 밤사이 수집/계산된 데이터를 기반으로 '오늘의 관심 종목' 리포트를 텔레그램으로 전송.
  2.  **점심 시간 (12:30):** TrendSurfer가 장중 수급과 지표가 좋아진 급등 예상 종목을 포착하여 김대리에게 알림.
  3.  **퇴근 후:** 웹 대시보드(Next.js)에 접속하여 포착된 종목의 상세 지표($RSI$, $MACD$ 등)를 확인하고 매매 전략 수립.

## 5. 기능 요구사항 (Functional Requirements)

### **P0 (MVP 핵심 기능 - v1.0)**
- **데이터 수집 및 저장 (ETL):**
  - [시스템]은 매일 장 마감 후 코스피/코스닥 전 종목의 기본 정보(종목코드, 종목명)와 일별 주가(OHLCV)를 수집하여 DB에 저장한다.
- **기술적 지표 산출:**
  - [시스템]은 수집된 주가를 바탕으로 보조지표($MA$, $RSI$, $Bollinger Bands$ 등)를 계산하여 저장한다.
- **장중 스크리닝 및 알림:**
  - [시스템]은 12:00~14:00 사이에 설정된 전략(예: 골든크로스)에 부합하는 종목을 필터링하여 사용자 메신저로 알림을 보낸다.
- **대시보드 (View):**
  - [사용자]는 웹 화면에서 종목별 차트와 계산된 지표 테이블을 조회할 수 있다.

### **P1 (High-Priority - v1.5 예상)**
- **모의 투자 (Paper Trading):** 실제 주문 전, 가상 매매를 통해 전략의 수익률을 시뮬레이션하는 기능.
- **커스텀 전략 설정:** 사용자가 웹 UI에서 $RSI < 30$ 등의 조건을 직접 조합하여 전략을 생성.

### **P2 (Nice to Have - v2.0 예상)**
- **실전 자동 주문 (Execution):** 증권사 API와 연동하여 조건 충족 시 실제 매수/매도 주문 실행.

## 6. 비기능 요구사항 (Non-Functional Requirements)
- **데이터 무결성:** 수집된 주가 데이터와 증권사 HTS 데이터 간의 오차 0.
- **성능:** 2,500개 종목 지표 계산 처리가 10분 이내에 완료될 것.
- **확장성:** 추후 미국 주식 데이터 추가가 용이한 DB 구조 설계.

## 7. 출시 계획 (Release Plan)
- **Phase 1 (분석/알림):** 데이터 파이프라인 구축 + 장중 알림 봇 + 조회용 웹 대시보드 오픈.
- **Phase 2 (실전 매매):** 키움증권 API 연동을 통한 자동 주문 모듈 탑재.
