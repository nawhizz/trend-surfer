# 전략 신호 스캔 조건 정리

일일 루틴의 마지막 단계인 **전략 신호 스캔**(`scripts/run_strategy.py` → `app/services/strategy_scanner.py`의 `StrategyScanner.scan()`)에서 사용하는 모든 조건을 단계별로 정리한 문서입니다.

> ⚠️ **중요:** 일일 스캔의 진입 조건은 백테스트의 `TrendFollowingStrategy`와 다릅니다.
> 백테스트는 `50EMA 기울기`·`ATR 과열 필터`·`지수 구조 필터`까지 사용하지만,
> 실제 일일 스캐너(`strategy_scanner.py`)는 더 단순한 3개 조건만 사용합니다.
> 아래 내용은 **실제 일일 스캔 기준**입니다. (백테스트와의 차이는 마지막 절 참고)

---

## 전체 흐름 (실행 순서)

```
1. 일봉 데이터 조회 (daily_candles)
        ↓
2. 유동성 필터 (거래대금 / 가격)
        ↓
3. 위험·부적격 종목 제외 (stocks 마스터)
        ↓
4. 기술적 지표 조회 (daily_technical_indicators)
        ↓
5. 신호 분석 (핵심 진입 3조건)
        ↓
6. 강도(strength) 계산 및 정렬
        ↓
   결과 출력 (콘솔 / 엑셀 / 텔레그램)
```

---

## 1단계: 일봉 데이터 조회

- `daily_candles` 테이블에서 대상 날짜(`target_date`, 기본값 = 전일)의 전종목 OHLCV를 조회
- 페이지네이션 단위: `BATCH_READ_PAGE = 1000`건
- 데이터가 **0건이면 즉시 빈 결과 반환** (휴장일 또는 데이터 미공시)

---

## 2단계: 유동성 필터 (`_apply_liquidity_filter`)

아래 두 조건을 **모두** 만족해야 통과합니다.

| 조건 | 기본값 | 상수 | 설명 |
|------|--------|------|------|
| **거래대금 ≥ min_amount** | **50억 원** (`5,000,000,000`) | `DEFAULT_MIN_AMOUNT` | `amount`가 없으면 `종가 × 거래량`으로 대체 계산 |
| **종가 ≥ min_price** | **1,000원** | `DEFAULT_MIN_PRICE` | 동전주 제외 |

- 실행 시 `--min_amount`, `--min_price` 인자로 조정 가능
- 동일 종목이 중복되면 첫 행만 사용

---

## 3단계: 위험·부적격 종목 제외 (`_should_exclude`)

`stocks` 마스터에서 종목 정보를 조회한 뒤, 아래 중 **하나라도** 해당하면 제외합니다.

1. **우선주** — `is_preferred = True`
2. **시장경보 종목** — `warning_type`이 존재 (관리종목·투자경고 등)
3. **종목명 키워드 포함** — `EXCLUDE_KEYWORDS`에 포함된 단어가 종목명에 있을 경우
   - `ETF`, `ETN`, `스팩`, `SPAC`, `관리종목`, `정리매매`, `투자위험`, `투자경고`, `거래정지`
4. **종목코드가 6자리 숫자가 아님** — ETF/ETN/SPAC 등 비정형 코드 배제

---

## 4단계: 기술적 지표 조회 (`_fetch_indicators`)

`daily_technical_indicators` 테이블에서 대상 날짜의 다음 지표를 조회합니다.

| 지표 | 설명 | 용도 |
|------|------|------|
| **MA(20)** | 20일 단순이동평균 | 진입 판정 |
| **HIGH(20)** | 20일 최고가 | 진입 판정 |
| **ATR(20)** | 20일 평균진폭 | 결과 표시(참고용) |
| **EMA_STAGE** | 이동평균 대순환 단계(1~6) | 결과 표시(참고용) |

---

## 5단계: 신호 분석 — 핵심 진입 조건 (`_analyze_signals`)

아래 **3가지를 모두 충족**해야 신호로 채택합니다.

| # | 조건 | 코드 | 의미 |
|---|------|------|------|
| 1 | **추세 상승** | `close > MA(20)` | 종가가 20일 이동평균 위 |
| 2 | **20일 신고가 돌파** | `close > HIGH(20)` | 종가가 직전 20일 최고가 돌파 |
| 3 | **양봉** | `close > open` | 당일 종가가 시가보다 높음 |

- `MA(20)` 또는 `HIGH(20)` 지표가 없는 종목은 건너뜀(`continue`)
- **ATR(20)·EMA_STAGE는 진입 판정에 사용하지 않으며**, 결과 표시용 메타데이터로만 포함

---

## 6단계: 강도(strength) 계산 및 정렬

- **강도(%) = ((종가 − 시가) / 시가) × 100** — 당일 장중 상승률
- 신호를 **강도 내림차순**으로 정렬

---

## 요약: 최종 선정되는 종목의 조건

아래 조건을 **모두 만족**하는 종목이 최종 신호로 선정됩니다.

- ✅ **유동성** — 거래대금 ≥ 50억 **AND** 종가 ≥ 1,000원
- ✅ **부적격 제외** — 우선주 아님 **AND** 시장경보 없음 **AND** 제외 키워드 없음 **AND** 6자리 숫자 코드
- ✅ **지표 존재** — MA(20)·HIGH(20) 지표가 모두 존재
- ✅ **진입 3조건** — 종가 > MA(20) **AND** 종가 > HIGH(20) **AND** 종가 > 시가

그리고 위 종목들을 **장중 상승률(강도) 높은 순**으로 정렬해 상위부터 보고합니다. (텔레그램은 상위 20개)

---

## 결과 출력

| 출력 대상 | 내용 |
|-----------|------|
| **콘솔** | ticker, close, strength, amount(억), MA(20), HIGH(20), ATR(20), STAGE, name |
| **엑셀** | `results/signal_YYYYMMDD.xlsx` (종목코드·종목명·종가·강도·거래대금·MA·HIGH·ATR·Stage) |
| **텔레그램** | 신호 리포트 발송 (`--no-notify` 시 생략) |

---

## 실행 명령어

```bash
cd backend
uv run ../scripts/run_strategy.py

# 옵션 예시
uv run ../scripts/run_strategy.py --date 2026-06-18 --min_amount 10000000000 --min_price 2000 --limit 20 --no-notify
```

| 인자 | 기본값 | 설명 |
|------|--------|------|
| `--date` | 오늘 | 대상 날짜 (YYYY-MM-DD) |
| `--min_amount` | 5,000,000,000 | 최소 거래대금(원) |
| `--min_price` | 1,000 | 최소 가격(원) |
| `--limit` | 0(전체) | 콘솔에 출력할 상위 N개 |
| `--no-notify` | - | 텔레그램 알림 생략 |

## 부록: 백테스트 전략과의 차이

일일 스캐너(`strategy_scanner.py`)와 백테스트 전략(`app/backtest/strategies/trend_following.py`)은 진입 조건이 다릅니다.

| 조건 | 일일 스캐너 | 백테스트 (TrendFollowing) |
|------|:-----------:|:-------------------------:|
| 종가 > 20일 신고가 | ✅ | ✅ |
| 종가 > MA(20) | ✅ | ❌ |
| 양봉 (종가 > 시가) | ✅ | ❌ |
| 50EMA 기울기 ≥ -0.2 | ❌ | ✅ |
| ATR 과열 필터 (ATR/종가 ≤ 15%) | ❌ | ✅ |
| 지수 시장 구조 필터 (EMA50) | ❌ | ✅ |

- 일일 스캐너는 **더 단순**하며, 변동성·시장 구조 필터를 적용하지 않습니다.
- 대신 백테스트엔 없는 **양봉 조건**과 **MA(20) 조건**을 추가로 사용합니다.
- 두 로직을 일치시키려면 별도 정렬 작업이 필요합니다.
