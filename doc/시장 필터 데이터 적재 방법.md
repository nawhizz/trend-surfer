# 시장 필터 데이터 적재 방법

추세추종 전략의 **시장 필터(Market Regime Filter)**가 사용하는 지수 데이터가 어떻게 수집·계산·저장되는지 정리한 문서입니다.

시장 필터는 KOSPI/KOSDAQ 지수의 추세 구조를 판단해, **지수 구조가 붕괴된 국면에서는 신규 진입을 막는** 역할을 합니다. 이 판단에 필요한 데이터가 DB에 적재되어 있어야 필터가 정상 동작합니다.

---

## 시장 필터가 사용하는 데이터

시장 필터(`backend/app/services/market_filter.py`)는 지수 2종 — **KOSPI(`KS11`)**, **KOSDAQ(`KQ11`)** — 의 데이터를 사용합니다. 두 지수의 데이터는 일반 종목과 **동일한 테이블**에 `ticker`로만 구분되어 저장됩니다.

| 데이터 | 저장 테이블 | 식별자 | 사용처(필터 메서드) |
|--------|------------|--------|---------------------|
| 지수 일봉 (OHLCV) | `daily_candles` | `ticker IN ('KS11','KQ11')` | `is_bullish()` — 종가 > 60MA 판정 |
| 지수 EMA50 기울기 | `daily_technical_indicators` | `indicator_type='EMA_SLOPE'`, `params={"period":50}` | `is_index_structure_ok()` — 구조 붕괴 판정 |

> ⚠️ **현재 `trend` 전략이 실제로 사용하는 조건은 `is_index_structure_ok()`(EMA50 기울기) 단독입니다.** MA60 기반 `is_bullish()`는 구현되어 있으나 2026-01-18 튜닝으로 주석 처리되어 미적용 상태입니다. (`trend_following.py` 참고)

---

## 적재 단계 1: 지수 마스터 등록 (`stocks` 테이블)

**진입점:** `index_collector.ensure_index_masters()` — `fetch_index_candles()` 호출 시 자동 선행 실행됨

- `KS11`, `KQ11`을 `stocks` 테이블에 **upsert**
- 등록 속성: `market='INDEX'`, `is_active=True`
- 지수 심볼 정의는 `index_collector.py`의 `INDEX_SYMBOLS`에 하드코딩

> 💡 지수가 `is_active=True`로 등록된다는 점이 **3단계 지표 계산에서 지수가 자동 포함되는 이유**입니다. (지표 계산기는 `is_active=True` 종목 전체를 순회)

---

## 적재 단계 2: 지수 일봉 수집 (`daily_candles` 테이블)

**진입점:** `scripts/backfill_index.py` → `index_collector.fetch_index_candles(start, end, ticker)`

1. **FinanceDataReader(FDR)**로 `KS11`/`KQ11`의 일봉 조회 (`fdr.DataReader`)
2. 행별로 가공하여 캔들 생성:
   - `open/high/low/close/volume` 매핑
   - `change_rate` = FDR `Change`(비율) × 100
   - 지수 특성상 `amount=0`, `market_cap=0`
   - 시가/종가가 `NaN`인 행은 스킵
3. `daily_candles` 테이블에 **배치 upsert**

**실행 예:**
```bash
cd backend
uv run ../scripts/backfill_index.py --start 2024-01-01 --end 2026-06-20
# 특정 지수만: --ticker KS11
```

---

## 적재 단계 3: 지수 지표(EMA_SLOPE) 계산 (`daily_technical_indicators` 테이블)

**진입점:** `scripts/run_daily_indicators.py` → `indicator_calculator.calculate_and_save_for_all_tickers()`

- `stocks` 테이블의 **`is_active=True` 종목 전체**를 순회 → 지수(`KS11`/`KQ11`)도 자동 포함
- 지수의 일봉 종가로 **EMA50 기울기**(`EMA_SLOPE`, period=50, ATR 정규화)를 계산
- `daily_technical_indicators` 테이블에 저장 (`indicator_type='EMA_SLOPE'`, `params={"period":50}`)

**실행 예:**
```bash
cd backend
uv run ../scripts/run_daily_indicators.py --date 2026-06-20
```

> ⚠️ 이 단계의 입력은 **2단계에서 적재된 당일 지수 일봉**입니다. 당일 지수 캔들이 없으면 당일 EMA_SLOPE가 계산되지 않습니다.

---

## 데이터 조회 — 필터 동작

시장 필터는 위에서 적재된 데이터를 다음과 같이 조회합니다.

```
market_filter.is_index_structure_ok(date, slope_threshold=-0.2)
   ├─ get_index_ema_slope('KS11', date)  → daily_technical_indicators 조회
   └─ get_index_ema_slope('KQ11', date)  → daily_technical_indicators 조회

   판정: KOSPI EMA50 기울기 ≥ -0.2  AND  KOSDAQ EMA50 기울기 ≥ -0.2
         → 둘 다 만족해야 진입 허용 (True)
         → 하나라도 None(데이터 없음)이면 진입 금지 (False)
```

> ⚠️ **데이터 미적재의 영향:** 당일 지수 일봉(2단계) 또는 EMA_SLOPE(3단계)가 없으면 `get_index_ema_slope`가 `None`을 반환하고, 필터는 **진입 금지(False)**로 동작합니다. 즉, 지수 데이터 누락은 "전 종목 신호 없음"으로 나타날 수 있습니다.

## 전체 데이터 흐름 요약

```
[지수 데이터 파이프라인]
   scripts/backfill_index.py
        └─ index_collector.fetch_index_candles()
             ├─ 1. ensure_index_masters() : KS11/KQ11 → stocks (market=INDEX, is_active=True)
             └─ 2. FDR 일봉 수집          : KS11/KQ11 → daily_candles           ★일봉 적재

   scripts/run_daily_indicators.py
        └─ indicator_calculator.calculate_and_save_for_all_tickers()
             └─ 3. is_active 종목 순회(지수 포함) → EMA_SLOPE 계산
                   → daily_technical_indicators (indicator_type=EMA_SLOPE)        ★지표 적재

[조회]
   market_filter.is_index_structure_ok()
        └─ daily_technical_indicators 에서 KS11/KQ11 EMA_SLOPE 조회 → 진입 허용/금지 판정
```

## ⚠️ 운영상 주의 — 일일 루틴과의 관계

현재 **`daily_routine.py`(일일 자동화 루틴)에는 지수 일봉 수집(2단계)이 포함되어 있지 않습니다.**

| 일일 루틴 단계 | 지수 일봉 적재 | 지수 EMA_SLOPE 계산 |
|----------------|:--------------:|:-------------------:|
| 3. 당일 시세 수집 (`run_collector --mode daily`) | ❌ (KRX API는 개별 종목만) | — |
| 4. 지표 계산 (`run_daily_indicators`) | — | ✅ (단, 당일 일봉이 있어야 함) |

- 3단계 당일 시세 수집은 **KRX 공식 API**로 개별 종목 OHLCV만 수집하며 `index_collector`를 호출하지 않습니다.
- 따라서 4단계는 지수를 처리 대상에 포함하지만, **그 입력인 당일 지수 일봉이 자동으로 들어오지 않으면** 당일 EMA_SLOPE가 비게 됩니다.
- 결과적으로 **지수 일봉은 `backfill_index.py`를 별도로 실행해 갱신**해야 합니다. 이것이 누락되면 시장 필터가 진입을 막아 신호가 나오지 않을 수 있습니다.

> 📌 **개선 후보:** `daily_routine.py`의 3단계(당일 시세 수집)와 4단계(지표 계산) 사이에 지수 일봉 수집 단계를 추가하면, 일일 루틴만으로 시장 필터 데이터가 완결됩니다.
